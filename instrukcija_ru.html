Инструкция по работе с личным кабинетом

Вход в систему → Раздел «Izveštaji» → Сводка «Rezime» и отчёт по продаже билетов на «Ђурологија» 17.04.2026

1
Открыть сайт и нажать кнопку «Prijava»

Перейдите на сайт polakohedonist.club/sr. В правом верхнем углу найдите и нажмите кнопку Prijava ▾.

💡 Кнопка «Prijava» — «Вход» на сербском. Находится в правом верхнем углу страницы.
2
Заполнить форму входа

Появится окно «Prijavite se na profil». Введите email и пароль, затем нажмите жёлтую кнопку Prijava.

🔑 Если забыли пароль — нажмите «Zaboravili ste lozinku?» для восстановления.
3
Вход выполнен — перейти в личный кабинет

После входа кнопка сменится на ● Profil с аватаркой. Нажмите на неё для перехода в кабинет.

✅ Жёлтый круг с буквой рядом с «Profil» означает успешный вход в систему.
4
Найти раздел «Izveštaji» в боковом меню

В личном кабинете найдите в левом меню пункт Izveštaji и нажмите на него.

📌 «Izveštaji» — это «Отчёты» на сербском. Раздел доступен только для менеджеров и администраторов.
5
Просмотр сводки «Rezime» по всем мероприятиям

В верхней части страницы «Izveštaji» расположена строка Rezime — общая сводка по всем мероприятиям. Нажмите иконку 📄 справа от неё, чтобы открыть сводный отчёт.

📊 Rezime — это сводный отчёт по всем вашим мероприятиям сразу. Ниже в списке «Događaji» — отдельные отчёты по каждому мероприятию.
6
Выбрать мероприятие из списка

В разделе «Događaji» найдите нужное мероприятие и нажмите жёлтую иконку 📄 справа от него.

📋 Нажмите 📄 рядом с нужным мероприятием. Для сводки по всем — нажмите 📄 в строке Rezime вверху.
7
Просмотр отчёта — «Polako Event Dashboard»

После нажатия на 📄 откроется дашборд с полной статистикой по мероприятию. Ниже описан каждый блок отчёта.

📊 Дашборд показывает полную картину продаж: от количества сгенерированных билетов до данных о каждом покупателе. Ниже — подробное описание каждого блока.
i
Блок 1 — «Event Info» (Информация о мероприятии)

Верхний блок показывает основные данные мероприятия: дату и время начала, и продолжительность.

📅 Start Date — дата и время начала мероприятия. Duration — плановая продолжительность в минутах.
i
Блок 2 — «Generated tickets for event» (Сгенерированные билеты)

Таблица показывает сколько билетов было создано для продажи по каждому типу и суммарно.

🎟 Этот блок показывает лимит продажи: сколько билетов было создано и доступно для покупки. Regular 200 означает, что доступно 200 обычных билетов.
i
Блок 3 — «Sold Active Tickets Counter» (Проданные активные билеты)

Круговая диаграмма (donut chart) показывает количество активных (не возвращённых) проданных билетов.

🔵 Sold Active Tickets Counter — показывает сколько билетов куплено и не возвращено. Сейчас 2 из 200 доступных билетов проданы и активны.
i
Блок 4 — «Refunded Tickets Counter» (Возвращённые билеты)

Счётчик показывает количество билетов, по которым был оформлен возврат.

↩ Refunded Tickets Counter — количество оформленных возвратов. Если равно 0 — все купленные билеты активны, возвратов не было.
i
Блок 5 — «Visited Tickets Counter» (Отсканированные билеты)

Показывает сколько билетов было отсканировано на входе при помощи приложения-сканера.

📲 Visited Tickets Counter — количество посетителей, чьи QR-коды были отсканированы на входе. 0 означает, что мероприятие ещё не началось или сканер не использовался.
i
Блок 6 — «Sold tickets for event» (График продаж по типам)

Столбчатая диаграмма показывает количество проданных билетов в разбивке по типам (тегам).

📊 Sold tickets for event — диаграмма продаж по типам билетов. Regular = 2 означает что продано 2 обычных билета. При наличии других типов (VIP, студенческий и т.д.) — для каждого будет свой столбец.
i
Блок 7 — «Revenue Currency Breakdown» (Финансовая разбивка)

Столбчатая диаграмма показывает финансовые потоки: сколько получает платформа и сколько возвращается организатору.

💰 Revenue Currency Breakdown — финансовая разбивка в разбивке по валютам. Platform Income = полная сумма от покупателей, Platform Fee = комиссия сервиса, Organizer Return = сумма к выплате вам как организатору.
i
Блок 8 — «Paid Tickets» (Список оплаченных билетов)

Детальная таблица с данными по каждому оплаченному билету — кто купил, когда, по какой цене и прошёл ли на мероприятие.

📋 Paid Tickets — детальный список каждого купленного билета. Колонки: Purchase Time — время покупки, Ticket Tag — тип билета, Ticket Price — цена, Organizer Return — ваш доход с этого билета, Platform Fee — комиссия сервиса, Is Visited — прошёл ли покупатель на мероприятие (No = не прошёл, Yes = прошёл).
